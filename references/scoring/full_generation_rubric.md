# full_generation 评分参考

适用场景：用户提供完整 HarmonyOS NEXT 工程、模块骨架或大段新生成代码，目标是评价其作为新工程产物的质量。

## 一级维度与权重

| 一级维度 | 权重 | 核心关注点 |
|---|---:|---|
| 工程结构与组织质量 | 20 | 目录结构、模块边界、公共能力组织 |
| 代码正确性与静态质量 | 20 | ArkTS/ArkUI 语法、类型安全、明显错误密度 |
| 架构与职责划分 | 20 | 页面/组件职责、状态与数据流、复用与抽象 |
| 可维护性与可读性 | 15 | 命名、复杂度、可演进性 |
| 平台规范符合度 | 15 | ArkUI 组织方式、ArkTS 约束、HarmonyOS 工程实践 |
| 风险控制与稳定性 | 10 | 稳定性、性能、安全边界 |

## 细化评分表

说明：二级指标按下表离散档位给分，默认不做插值；若证据不足，可降低置信度并进入人工复核，但不要用“猜测”补足分数。

### 工程结构与组织质量

| 二级指标 | 分值 | 给分标准 |
|---|---:|---|
| 目录结构合理性 | 7 | 目录分层自然，页面、组件、状态、服务、工具、资源等落位稳定，无明显模板残留，后续扩展路径清晰。 |
| 目录结构合理性 | 5 | 主体结构合理，少量文件落位不够理想或存在轻微过度拆分，但整体仍容易接手。 |
| 目录结构合理性 | 3 | 基本能看懂，但存在多处职责错位、目录命名含糊或脚手架残留，扩展路径不够稳定。 |
| 目录结构合理性 | 1 | 目录组织混乱，文件落位大量不合理，阅读者难以快速判断模块归属。 |
| 目录结构合理性 | 0 | 目录结构严重失真，基本无法作为可接手工程组织。 |
| 模块边界清晰度 | 7 | 模块职责边界明确，跨模块依赖关系克制，分层方向稳定，不依赖隐式耦合。 |
| 模块边界清晰度 | 5 | 主要边界清楚，仅有少量交叉依赖或职责粘连，但不影响整体理解。 |
| 模块边界清晰度 | 3 | 若干模块职责重叠，页面层、状态层、工具层之间边界模糊，后续演进成本偏高。 |
| 模块边界清晰度 | 1 | 模块耦合明显，跨层调用随意，职责划分失效。 |
| 模块边界清晰度 | 0 | 基本不存在有效模块边界，代码呈现大块堆叠或任意串联。 |
| 公共能力组织质量 | 6 | 通用能力抽取位置正确，复用路径清楚，既不散落在页面层，也不过度包装。 |
| 公共能力组织质量 | 4 | 已有一定抽取与复用，但仍存在局部重复实现或公共层轻微臃肿。 |
| 公共能力组织质量 | 2 | 公共逻辑散落、多处复制粘贴，或把本应局部的逻辑错误上提为公共层。 |
| 公共能力组织质量 | 0 | 公共能力组织明显失控，复用关系和职责归属都不成立。 |

### 代码正确性与静态质量

| 二级指标 | 分值 | 给分标准 |
|---|---:|---|
| ArkTS/ArkUI语法与类型安全 | 8 | 无明显 ArkTS/ArkUI 语法错误，类型声明完整，空值、异步、接口返回值等关键位置处理稳定。 |
| ArkTS/ArkUI语法与类型安全 | 6 | 存在少量局部类型宽松、可空处理不够严谨或类型推断依赖过强，但不构成成片错误。 |
| ArkTS/ArkUI语法与类型安全 | 4 | 有多处类型或语法层面的可疑点，需要人工补修后才能稳定接手。 |
| ArkTS/ArkUI语法与类型安全 | 2 | 明显存在较多类型错误、未定义引用、导入导出不稳或 ArkTS 约束冲突。 |
| ArkTS/ArkUI语法与类型安全 | 0 | 高密度静态错误或明显不可运行片段，接近触发 G1/G2。 |
| 静态坏味道控制 | 6 | 重复逻辑、死代码、魔法值、过长方法、无意义包装等坏味道控制良好，无明显积累。 |
| 静态坏味道控制 | 4 | 有少量坏味道，但局部且可接受，不影响整体质量判断。 |

| 静态坏味道控制 | 0 | 代码坏味道密集出现，已明显影响可信度和可维护性。 |
| 明显错误密度 | 6 | 几乎看不到明显错误点，偶发问题不形成模式。 |
| 明显错误密度 | 4 | 有少量单点错误，但集中修补成本较低，不影响整体结构判断。 |
| 明显错误密度 | 2 | 多个文件或关键链路出现明显错误，整体可信度下降。 |
| 明显错误密度 | 0 | 错误呈片状分布，工程整体可用性很低。 |

## 本类型常见风险
- 直接照搬 Web / React / Vue 组织方式。
- ArkUI 页面结构失真，UI 与状态杂糅。
- 生成大量“看起来很多”但不可接手的模板代码。
- 公共能力抽取位置混乱，后续扩展困难。

## 输出时应强调
- 工程能否作为“高置信的新工程产物”接手。
- 哪些问题属于结构性问题，而不是局部瑕疵。
- 是否触发 G1 / G2 / G3 硬门槛候选。
